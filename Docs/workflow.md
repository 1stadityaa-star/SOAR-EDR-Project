# Workflow

## Overview

This SOAR workflow automates the response to security detections generated by LimaCharlie. When a malicious process is detected, Tines receives the alert, notifies the analyst through Slack and email, requests analyst approval, and isolates the affected endpoint if approved.

# Node Explanation

## 1. Retrieve Detection (Webhook)

**Purpose**

Receives detection events from LimaCharlie whenever a Detection & Response (D&R) rule is triggered.

**Input**

- Detection Title
- Hostname
- Username
- Source IP
- File Path
- Command Line
- Sensor ID
- Detection Link

---

## 2. Slack Alert

**Purpose**

Immediately notifies the SOC team in Slack about the detection.

**Message includes**

- Detection Title
- Computer Name
- User
- Source IP
- File Path
- Detection Link

---

## 3. Email Alert

**Purpose**

Sends an email containing the detection details for documentation and analyst review.

---

## 4. User Prompt

**Purpose**

Displays a decision page asking the analyst whether the endpoint should be isolated.

Options:

- ✅ Yes
- ❌ No

---

## 5. Yes Condition

If the analyst selects **Yes**, the workflow continues to isolate the endpoint.

---

## 6. HTTP Request – Isolate Sensor

**Purpose**

Calls the LimaCharlie REST API using JWT authentication to isolate the compromised endpoint.

Method

POST

Authentication

Bearer JWT

---

## 7. Slack Confirmation

After successful isolation, a Slack message confirms that the endpoint has been isolated.

Example:

```
✅ Endpoint successfully isolated.

Hostname: windowsa
User: adi
Detection: Adi - HackTool - Lazagne
```

---

## 8. No Condition

If the analyst selects **No**, the endpoint is not isolated.

---

## 9. Slack Notification

Sends a Slack message informing the SOC team that isolation was declined by the analyst.

Example:

```
⚠ Analyst chose not to isolate the endpoint.
```

---

# Overall Workflow

1. LimaCharlie detects malicious activity.
2. Detection is sent to Tines.
3. Slack notification is generated.
4. Email notification is generated.
5. Analyst reviews the alert.
6. If **Yes**
   - Endpoint is isolated.
   - Slack confirms isolation.
7. If **No**
   - Endpoint remains active.
   - Slack records that isolation was declined.
